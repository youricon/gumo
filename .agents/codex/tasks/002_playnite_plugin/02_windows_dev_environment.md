# Task 02: Windows Dev Environment

## Status

Not started

## Goal

Make Windows the primary, documented environment for Playnite plugin development and debugging.

## Depends On

- 01 Repo And Project Setup

## Deliverables

- documented Windows toolchain requirements
- documented Windows VM setup
- documented Playnite installation and debugging workflow
- documented connectivity workflow between Windows Playnite and Gumo

## Steps

- Document required Windows tools and SDKs.
- Document a recommended Windows VM setup.
- Document how to attach the plugin to a Playnite installation for development.
- Document how to reach a Gumo server running on the NixOS host.
- Document the normal daily plugin development loop.

## Acceptance Criteria

- A developer can prepare a Windows environment without guessing.
- The Windows workflow covers build, debug, and runtime verification.
- The Playnite-to-Gumo connectivity path is explicit.

## Tracking Checklist

- [ ] Toolchain documented
- [ ] VM workflow documented
- [ ] Playnite debug workflow documented
- [ ] Host/VM connectivity documented

## Notes

- Windows is the authoritative plugin environment.
